## 2024-05-13 - Icon-only buttons accessibility using Radix Tooltip
**Learning:** In components leveraging shadcn/ui and Radix UI Tooltips, it is crucial to wrap icon-only buttons (or badges) with `Tooltip` components instead of relying solely on native HTML `title` attributes. This provides better cross-browser consistency and explicit ARIA associations. Moreover, the top-level layout usually includes a `TooltipProvider`, but when testing these components in isolation with React Testing Library, `TooltipProvider` MUST be explicitly added to the `render` wrapper.
**Action:** Always wrap icon-only elements in a Radix Tooltip to ensure proper accessible labels and visual tooltips. In unit tests, wrap the component in `<TooltipProvider>` and verify tooltip text using `userEvent.hover()` combined with `waitFor(() => expect(screen.getByRole("tooltip")).toHaveTextContent("..."))`.
